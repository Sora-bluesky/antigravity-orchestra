# /checkpoint - セッション永続化ワークフロー

## 使用方法

```
/checkpoint              # 基本: 会話履歴とタスク状態を保存
/checkpoint --full       # 完全: git履歴・ファイル変更も含めて保存
```

## Step 1: 状態の収集（Antigravity）

以下の情報を収集：

- 現在のタスク状態
- 完了した作業
- 未完了の作業
- 重要な決定事項

## Step 2: チェックポイントファイル作成

`docs/checkpoints/{timestamp}.md` に保存。

**フォーマット**:

```markdown
# Checkpoint: {日時}

## 完了した作業
- {作業1}
- {作業2}

## 進行中の作業
- {作業}: {進捗状況}

## 未着手の作業
- {作業}

## 重要な決定事項
- {決定1}: {理由}

## 次回再開時の注意点
- {注意点}

## 関連ファイル
- {ファイルパス}
```

## Step 3: --full オプション時の追加処理

- git status の結果を記録
- 変更されたファイルのdiffを記録
- 環境情報を記録

## Step 4: 確認

チェックポイントが正しく保存されたことを確認し、ユーザーに報告。
