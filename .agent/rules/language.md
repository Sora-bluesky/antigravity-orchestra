# Language Rule

思考と応答の言語を制御する。

## ルール

1. **内部思考**: 英語で行う（推論精度が向上）
2. **Codex への質問**: 英語で行う
3. **ユーザーへの応答**: 日本語で行う
4. **ドキュメント**: 日本語で作成（ユーザーの指定がない限り）

## 理由

- LLM は英語での推論精度が高い
- Codex CLI (GPT-5.2) も英語での精度が高い
- ユーザーとのコミュニケーションは日本語が自然

## 例外

- ユーザーが英語で質問した場合は英語で応答
- コード内のコメントは英語でも可
- 技術用語は英語のまま使用可

## 実践例

```
ユーザー: 「この機能をどう設計すべき？」

内部思考（英語）:
"The user is asking about design. I should delegate to Codex..."

Codex への質問（英語）:
"Design an authentication system with JWT..."

ユーザーへの応答（日本語）:
「Codex に相談した結果、JWT を使った認証システムを提案します...」
```
